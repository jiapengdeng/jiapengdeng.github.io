什么是路由

这个概念在我没有理清之前，各种短语，关键字在我脑中混乱的游荡，什么navigator， history，hash，路由器呀，一团糟，但是还好，我能知道走某个链接就能跳进某个方法中去执行，幸运的是，这个规律让我在开发中几乎没有遇到什么问题（现在我是个前端），很显然，这个知识点，我是该总结下，把条理理顺了。

网路了一圈，发现一个能看得懂的回答：

网络原理中，路由指的是根据上一接口的数据包中的IP地址，查询路由表转发到另一个接口，它决定的是一个端到端的网络路径。

明白了一点（这勾起了我对《信息技术基础》的怀念），这是物理层的路由概念，哪web路由又是什么呢？

应用程序捕获到请求资源，对请求携带的资源的进行解析，根据解析到了结果做响应的处理，这个过程就是路由，往简单了说就是 “路径判断”。
–引自我的强行总结

路由能做什么

这个问题我还是顺着 “路径判断” 的理解来回答吧:

路径存在于报文的第一行的第一部分， GET /path?foo=bar http/1.1, 当接收到这个路径后， 就可以做if判断了。如果你高级一点的话可以做一个分发控制器，分发到应用的特定程序片段。

对于这种route操作，npm仓库中已经有很多第三方包可以供你使用了，你无需劳心劳神的重复造轮子了。

koa-route是什么

koa-route就是第三方route包的其中一个，她是一个带有koa特性的路径匹配工具。

koa-route有什么特点

  • 保留Express路由的使用方式： app.get, app.put, app.post等
  • 申明式式URL的参数和正则匹配。
  • 字符串或者正则表达式的路由匹配
  • 用URL 构造器申明路由
  • 用允许的方法响应 Options 请求
  • 支持 405,和504
  • 多个路由中间件
  • 多个路由

API

学习一个模块，目的是服务于需求，但是怎么才能让你避免走进错误的胡同呢？我觉得最直接的还是看模块的API，如果他所罗列出的接口，方法都不能满足你的需求，那么还是放弃这个模块，早做其他打算。

Router

Router是暴露给用户的一个类，通过new Router([option]) 可以得到一个路由实例。option不用传。

基础用法

router.get|put|post|patch|delete

创建一个请求方法。普通使用者get和post就够使用，但是非要严格的按照http规范请求的话（如rest），那么所有http.METHODS支持的请求方式都可作为上上面的动词，但要注意，如果是delete，那么用del代替。
