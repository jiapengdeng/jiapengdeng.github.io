koa运行原理

发表于 2016-07-25

本文会从es6语法，koa源码入手，浅层次的理解koa的运行机制。

koa是一个比express更小、更健壮、更富有表现力的 Web 框架。使用 koa 编写 web 应用，通过组合不同的 generator，可以免除重复繁琐的回调函数嵌套，并极大地提升常用错误处理效率。

koa确实是一个有前途的node web框架，他的理念确实为解决某些问题指明了方向，但是他的代码有时看起来让人费解，让我这个初学者对他望而生畏，但是栗子好吃，总不会因为它身上的刺放弃美味了吧。下面浅薄的记录了对koa的学习与理解

矛与盾

高并发

同样是web服务器，为什么 node能比 tomcat，apache拥有更高的响应能力，主要是集中了V8的执行效率与异步IO模型的优势而来的，对于cpu密集型计算，node的性能是仅次于c语言的。而对于io密集型的计算，更是node强项，至于原因，node的异步IO+事件循环可以避免进程或线程间切换时的性能损耗。更细致的也没必要解释了，可以看朴灵的《深入浅出nodejs》。

语法基础

generator

generator形式上和普通函数一样, 都是 function name(){}的形式，但又有所区别：

  • name 与 function 之间有个 *,即 function *name(){}
  • 函数体用yield或return返回
  • 返回结果是一个具有属性value和done的对象。value对应返回的值，done对应的是遍历器有没有遍历完。
  • 函数块被yield分割成多块，调用函数返回的是一个遍历器,通过调用遍历器的next执行程序块
  • yield 起到暂停后面执行函数的作用，等到下一次调用next才会继续往下执行，而且在暂停的同时返回紧跟在yield后面的表达式的值。

promise

所谓Promise，简单说就是一个容器，里面保存着某个未来才会结束的事件（通常是一个异步操作）的结果。从语法上说，Promise是一个对象，从它可以获取异步操作的消息。>Promise提供统一的API，各种异步操作都可以用同样的方法进行处理。

如果要在通俗点，Promise就是一个“承诺”，并且还是一个信用度很高的“承诺”，只有本身确定，不受外界干扰而改变“承诺”，并且承诺一旦履行，就不再改变。例如明天我承诺会给你答复要不要去你家，这个承诺一定会履行，但是至于答复去还是不去，去就对应resolve,不去对应reject。

如果想要捕获到结果, 就用.then去捕获。

把所有处理交给第三方组件，也就是koa中间件，使自己更轻量和更高效。
